Full-stack health prediction app using a **locally trained ML model**.
**No internet or external API required after install.**

## ⚠️ Python 3.14 on Windows — Installation Fix

You saw this error because pip tried to **compile scikit-learn from source**,
which requires a C compiler (Visual Studio Build Tools) that you don't have.

**Root cause:** Old pip + old scikit-learn versions have no pre-built wheel for
Python 3.14. scikit-learn ≥ 1.8 and numpy ≥ 2.0 do ship cp314 wheels for
Windows — so the fix is to install the right versions the right way.

### ✅ Option A — Use the setup script (recommended, one click)
```bat
cd backend
setup_windows.bat
```
This script upgrades pip, creates a venv, installs everything with
`--only-binary :all:` (forces pre-built wheels, zero compilation), and trains
the model. Done.

### ✅ Option B — Manual steps (PowerShell / cmd)
```bat
cd backend

:: 1. Upgrade pip FIRST (old pip chooses wrong wheels)
python -m pip install --upgrade pip

:: 2. Create + activate virtual environment
python -m venv venv
venv\Scripts\activate

:: 3. Install packages — only-binary prevents any source builds
pip install --only-binary :all: "numpy>=2.0.0"
pip install --only-binary :all: "scikit-learn>=1.8.0"
pip install --only-binary :all: "pandas>=2.2.0"
pip install joblib flask==3.0.3 flask-sqlalchemy==3.1.1 flask-cors==4.0.1

:: 4. Train the model + start server
python train_model.py
python app.py
```

### ✅ Option C — Use Conda (avoids all wheel issues entirely)
```bat
conda create -n mira python=3.14
conda activate mira
conda install scikit-learn numpy pandas
pip install flask flask-sqlalchemy flask-cors joblib
python train_model.py
python app.py
```

## Quick Start

### Windows (Python 3.14)
```bat
cd backend
setup_windows.bat        ← installs
python train_model.py        ← Trains

venv\Scripts\activate
python app.py            ← http://localhost:5000

:: New terminal:
cd frontend && npm install && npm start   ← http://localhost:3000
```

### macOS / Linux
```bash
cd backend && bash setup_unix.sh
source venv/bin/activate && python app.py

# New terminal:
cd frontend && npm install && npm start
```

## Patient Fields

| Field | Validation |
|---|---|
| Full Name | Required, ≥ 2 characters |
| Date of Birth | Required, must be in the past |
| Email Address | Required, valid format, unique per patient |
| Glucose (mg/dL) | Required, positive; normal 70–100 |
| Haemoglobin (g/dL) | Required, positive; normal 12–17 |
| Cholesterol (mg/dL) | Required, positive; normal < 200 |
| Remarks | Auto-generated by ML model (read-only) |

## Health Risk Categories

| Label | Trigger |
|---|---|
| Healthy | All values within normal range |
| Diabetes Risk | Glucose > 140 mg/dL |
| Anaemia Risk | Haemoglobin < 12 g/dL |
| Dyslipidaemia Risk | Cholesterol > 200 mg/dL |
| High Composite Risk | Two or more abnormal values |

---

## ML Model

- **Algorithm:** VotingClassifier (soft) — RandomForest (300 trees) + GradientBoosting (200)
- **Pipeline:** StandardScaler → VotingClassifier
- **Features:** 9 engineered features (raw values, ratios, normalised scores, composite count)
- **Training data:** ~2,150 synthetic samples with Gaussian noise from clinical reference ranges
- **Accuracy:** ~98–99% on test set
- **Fallback:** Clinical threshold rules if model.pkl is missing

## Bug Fixes

**Single-character input / Add Patient broken:**
`FormField` was defined inside `PatientForm`'s render function. React treated it
as a brand-new component on every render, unmounting and remounting the `<input>`
after each keystroke. Fixed by moving `FormField` to module scope, adding
`useCallback` on the change handler, and a `key` prop on `<PatientForm>`.
